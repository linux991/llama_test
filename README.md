# Horror Fine-Tuning Detection Experiment

This repository prepares a Colab-ready replication of the experiment from
*When Detection Fails: The Power of Fine-Tuned Models to Generate Human-Like
Social Media Text*, adapted from social media posts to English horror stories.

The planned GPU experiment compares:

- human-written horror fragments;
- fragments generated by base `meta-llama/Meta-Llama-3-8B-Instruct`;
- fragments generated by the same model after QLoRA fine-tuning.

The CPU-only local pipeline prepares the corpus, split, prompts, and Colab
payload before any GPU work starts.

## Repository Layout

```text
scripts/
  extract_creepypastas.py      # XLSX/CSV/JSON/TXT -> one .txt file per story
  prepare_colab_payload.py     # split, chunks, prompts, reports, payload zip

reports/
  corpus_preparation_report.md # generated corpus/split/leakage summary

horror_qlora_detection_experiment_colab.ipynb
horror_experiment_for_dummies_guide.md
```

Local data artifacts are intentionally ignored by git:

```text
creepypastas.xlsx
data/
models/
outputs/
```

## Local Preparation

Create and activate a virtual environment:

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
python -m ipykernel install --user --name horror-experiment --display-name "Horror Experiment"
```

Put the source workbook at:

```bash
creepypastas.xlsx
```

Extract cleaned stories:

```bash
python3 scripts/extract_creepypastas.py \
  --input creepypastas.xlsx \
  --output-dir data/creepypasta_stories_txt \
  --overwrite
```

Build the Colab payload:

```bash
python3 scripts/prepare_colab_payload.py
```

Validate scripts and notebook JSON:

```bash
python -m py_compile scripts/extract_creepypastas.py scripts/prepare_colab_payload.py
python -m json.tool horror_qlora_detection_experiment_colab.ipynb > /tmp/horror_notebook_validated.json
```

Default payload settings:

- story-level split: 80% train, 20% eval;
- chunk length: 700-1500 characters;
- payload train chunks: 2000;
- payload eval chunks: 500;
- exact train/eval chunk overlaps removed from eval.

Generated payload:

```text
data/horror_experiment_payload.zip
```

The payload contains:

```text
train_with_descriptions.csv
eval_with_descriptions.csv
split_metadata.csv
corpus_summary.csv
corpus_preparation_report.md
```

## Colab Run

Upload `horror_qlora_detection_experiment_colab.ipynb` and
`data/horror_experiment_payload.zip` to Google Colab.

The final notebook should:

1. install dependencies;
2. load the prepared payload;
3. inspect train/eval split statistics;
4. load `meta-llama/Meta-Llama-3-8B-Instruct`;
5. generate base AI horror fragments from eval descriptions;
6. fine-tune Llama 3 8B with QLoRA on train descriptions/fragments;
7. generate fine-tuned AI fragments from the same eval descriptions;
8. compare human/base/fine-tuned text with stylometry and a supervised detector.

Llama 3 8B requires Hugging Face access. Accept the model license and provide
`HF_TOKEN` in Colab secrets or run `notebook_login()`.
